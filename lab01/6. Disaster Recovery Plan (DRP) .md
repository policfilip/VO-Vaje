**Purpose**

This Disaster Recovery Plan describes the procedures and responsibilities required to recover IT systems and data after a disruption such as cyber attacks, hardware failures, or natural disasters.

The goal is to minimize downtime and data loss while ensuring business continuity.

**Disaster Scenarios**  
Possible disasters include:

* Cyber attack (ransomware)  
* Hardware failure  
* Data corruption  
* Network outage  
* Power outage  
* Natural disaster (fire, flood)

**Recovery Procedures**

**8.1 Server Failure**

Steps:

Identify failed server  
Activate backup infrastructure  
Restore latest backup  
Verify system integrity  
Reconnect users

**8.2 Ransomware Attack**

Steps:

Disconnect infected systems  
Identify affected systems  
Restore from clean backup  
Reset passwords  
Conduct forensic analysis

**8.3 Data Loss**

Steps:

Identify missing data  
Retrieve backup  
Restore database  
Verify data integrity

**8.4 Network Outage**

Steps:

Identify network failure  
Check firewall and router status  
Restart network equipment if required  
Restore network connectivity  
Verify access to critical systems

**8.5 Power Outage**

Steps:

Confirm power outage and notify facility management  
Switch to UPS / backup power if available  
Safely shut down critical systems if needed  
Restore systems after power returns  
Verify system functionality

**8.6 Natural Disaster (Fire, Flood)**

Steps:

Ensure personnel safety and evacuate if required  
Assess damage to IT infrastructure  
Activate disaster recovery plan  
Restore systems from offsite backups  
Verify system availability and data integrity
